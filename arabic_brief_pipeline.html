System Architecture · Arabic Book Brief Engine

Automated Arabic Book
Brief Pipeline

A production-grade, end-to-end system for transforming Arabic books into polished 3–5 minute audio/video summaries — from raw PDF to broadcast-ready output.

4 Phases Multi-Agent LLM Arabic-First Design
Pipeline Architecture
1
Phase 1 Extraction & Pre-processing
📄

1a — PDF Ingestion & Type Detection

Detect whether the PDF is digitally born or scanned before choosing the extraction path. Use pdfminer.six or PyMuPDF for digital PDFs. For scanned books use EasyOCR (Arabic model) or Tesseract 5 with ara tessdata — both outperform Cloud Vision for Arabic at scale and cost.

🔪

1b — Semantic Chunking

Do not send the full book to the LLM. Use chapter/section boundaries from the PDF outline first, then fall back to paragraph-level chunking with a 1500-token max and a 200-token overlap. Libraries: langchain.text_splitter.RecursiveCharacterTextSplitter with Arabic-aware sentence separators (['.\n', '،\n', '\n\n']).

🔤

1c — Diacritization (Harakat)

Run text through Farasa Diacritizer (QCRI — most accurate for MSA) before any downstream use. This is a pipeline-level requirement, not optional, as it directly governs TTS pronunciation quality. Alternative: mishkal (open-source, slower). Store both raw and diacritized versions.

⚠ Arabic-Specific Pitfall: PyMuPDF often extracts Arabic text in visual (right-to-left reversed) order. Apply python-bidi + arabic-reshaper immediately after extraction before any further processing.
2
Phase 2 Multi-Agent Script Generation
🗺️

2a — Analyst Agent (Map-Reduce)

Before writing, extract structured meaning from all chunks: themes, key arguments, memorable quotes, author's tone. Use a map-reduce pattern — summarize each chunk independently, then reduce to a unified "book DNA" JSON object passed to the next agent.

✍️

2b — Scriptwriter Agent

Input is the "book DNA" JSON, not raw text. Output: a 600–750 word Arabic script (Arabic TTS pace ≈ 120–140 wpm → 4.5–6 min at 750 words; target 680 words for 5 min).

أنت كاتب سيناريو محترف. بناءً على تحليل الكتاب التالي: {book_dna_json} اكتب سكريبت فيديو 680 كلمة باللغة العربية الفصحى يشمل: — هوك قوي في الجملة الأولى (سؤال أو إحصاء صادم) — 3 رؤى محورية مع انتقالات واضحة — خاتمة عاطفية تدفع المشاهد للتفكير — أسلوب {genre_tone} أخرج السكريبت فقط بدون أي شرح.
✅

2c — Editor Agent (Scored Review)

A second LLM call evaluates the script against a rubric and returns a structured JSON score: {"hook": 0-10, "flow": 0-10, "word_count": int, "grammar_issues": [...], "approved": bool}. If approved: false, the script loops back to 2b with the feedback injected. Max 2 retries before human escalation flag.

Orchestration: Use LangGraph for this multi-agent graph — it provides native support for conditional edges (the retry loop), state persistence, and streaming visibility into which agent is running.
3
Phase 3 Audio Synthesis (TTS)
🎙️

Voice Selection Strategy

Use the diacritized script (from Phase 1c) as input, not the raw script. Route to the best TTS engine based on your quality/cost trade-off:

🎛️

Post-Processing

After TTS, apply lightweight audio normalization using pydub: normalize loudness to –16 LUFS (podcast standard), add 500ms silence at section breaks, export as 128kbps MP3 for video, 256kbps AAC for audio-only distribution.

Phase 4 Video Assembly

💡

Recommended Path for Launch

Start with Track 2 (Kinetic Text). It is fully self-hosted, costs nothing beyond compute, produces a distinctive aesthetic, and handles RTL Arabic natively with arabic-reshaper + Pillow. This is also a strong differentiator — most Arabic book content channels look the same with avatar videos.

Arabic RTL in MoviePy: Pillow does not handle Arabic shaping natively. You must reshape + bidi-reorder every text string before passing it to ImageClip. Use: arabic_reshaper.reshape() → get_display() → then render.

Risk Register
LLM Hallucination in Summary

The editor agent must cross-check named claims against the extracted chunk text. Add a "factual grounding" check: any proper noun in the script must appear in the source chunks.

Arabic Dialect vs MSA Mismatch

Classify the book's register first (MSA, Egyptian, Levantine, etc.) using a lightweight classifier. Pass register as a parameter to both the scriptwriter and TTS voice selector.

TTS Cost at Scale

Cache audio by script hash. If the same book is re-processed (e.g. after an editor revision), retrieve cached audio rather than regenerating. Use Google Neural2 as cost-efficient fallback for draft passes.

Copyright / Rights

Ensure the pipeline operates only on books within legal rights window (public domain, licensed, or owned content). Add an input-validation step that flags PDFs without clear provenance metadata.
